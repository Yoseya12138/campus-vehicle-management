# 数据库权限配置建议

正式使用时，数据库集合不要开放给小程序前端直接读写。推荐原则：

```text
students        前端不可直接读写；通过云函数访问
teachers        前端不可直接读写；通过云函数访问
vehicles        前端不可直接读写；通过云函数访问
officers        前端不可直接读写；仅云函数访问
scanLogs        前端不可直接读写；仅云函数写入和管理员查询
violations      前端不可直接读写；通过云函数访问
rateLimits      前端不可直接读写；仅云函数访问
securityEvents  前端不可直接读写；仅云函数访问
```

如控制台支持自定义安全规则，可将集合权限设为：

```json
{
  "read": false,
  "write": false
}
```

如果需要学生本人查看自己的车辆，仍建议通过云函数 `myVehicles` 返回脱敏后的数据，不建议前端直接查数据库。

## 云存储权限

车辆照片、违停照片建议分目录：

```text
vehicle-photos/
violation-photos/
```

原则：

- 上传通过云函数或受控上传策略生成路径。
- 文件 ID 写入数据库前由云函数校验归属。
- 读取车辆照片时由云函数判断权限后返回临时访问链接。
- 违停照片仅保卫科和管理员可查看。

## 管理员初始化

首次部署后使用 `bootstrapAdmin` 初始化第一个管理员。初始化完成后：

1. 删除或更换 `BOOTSTRAP_CODE`。
2. 确认 `officers` 中只有授权人员。
3. 后续人员变更全部通过 `manageOfficer` 记录安全事件。
